# DockStart V0.9.6 UI 整合方案

## 目标

把“运行对接”页面已经形成的仪器工作台体验扩展到完整流程，同时减少页面跳转和重复信息。此方案不新增科学能力、不修改 Vina 行为，也不改变现有项目数据结构。

## 信息架构

主导航从分散页面收敛为四个工作区：

1. **项目**：项目总览、创建与打开项目、最近项目和当前阻塞项。
2. **结构准备**：结构获取、自动准备、PDBQT 导入整合为同一工作区的三个入口；根据 Basic/Assisted 模式只展开当前需要的路径。
3. **对接工作台**：搜索范围、Vina 参数、运行前检查与执行整合为连续工作台；3D 视图保持主画布，右侧放可操作项，下方放参数和状态仪表。
4. **结果**：结合能表格、构象查看、运行日志与 Markdown 报告整合为同一分析工作区。

工具链、设置、帮助保留为辅助入口，移出主流程步骤，不再与对接任务争夺导航层级。

## 统一页面骨架

每个主工作区都使用相同骨架：

- 深蓝色应用壳与紧凑的项目状态栏；
- 单一页面标题、当前任务说明和明确的主操作；
- 中央任务画布，不重复显示项目摘要；
- 右侧上下文栏，只放当前可操作信息、校验和下一步；
- 底部粘性操作区，固定“上一步 / 保存 / 继续”；
- 空状态、进行中、成功、警告和错误使用统一组件与留白规则。

## 页面去留

| 当前页面 | 处理方式 | 目标位置 |
| --- | --- | --- |
| 项目总览、创建/打开项目 | 合并 | 项目工作区 |
| 获取结构、准备结构、导入 PDBQT | 合并为模式化入口 | 结构准备工作区 |
| 设置搜索范围、Vina 参数、生成配置、运行前检查、执行对接 | 合并为分阶段工作台 | 对接工作台 |
| 结果、3D 构象、实验记录 | 合并 | 结果工作区 |
| 工具检测、工具链状态 | 合并 | 工具链辅助页 |
| 设置、帮助 | 保留 | 顶栏辅助入口 |

## 实施顺序

### 阶段 0：稳定 V0.9.5

- 完成版本与发布说明同步；
- 清理已删除 Viewer 的残留状态和路由；
- 补齐全屏和构象弹窗的键盘、焦点、滚动锁定；
- 对四个核心流程做真实桌面端截图与回归检查。

### 阶段 1：建立共享工作台组件

- 提炼 Run 页面中的 Hero、主画布、右栏、底部操作区和状态组件；
- 建立统一密度、间距、字号、表单、卡片和状态色规则；
- 先替换页面外壳，不改业务逻辑。

### 阶段 2：合并结构准备

- 用 Basic / Assisted 两条路径驱动内容显隐；
- 避免“获取、准备、导入”三个页面同时出现在主导航；
- 保留每一步的错误恢复与原始日志。

### 阶段 3：合并对接工作台

- 把 Box、参数、配置和运行变为同一页面的连续阶段；
- 3D 画布保持视觉中心；
- 参数更改、校验状态和运行按钮在同一上下文内反馈。

### 阶段 4：合并结果与报告

- 结果表格作为主画布；
- 构象查看使用可扩展详情区或大尺寸对话框；
- 日志与 Markdown 实验记录作为同一运行的次级标签页。

## 验收标准

- 新用户从创建项目到开始对接，主导航不超过四个工作区；
- 任一页面只有一个视觉主操作；
- 结构准备和对接设置不重复展示同一项目摘要；
- 键盘可打开、操作并关闭所有弹窗和扩展视图；
- 1280×720 下无横向溢出，主要操作无需滚动到不可见区域；
- 页面切换不重复触发可避免的后端状态请求；
- Basic、Assisted、Demo 三种模式均可沿现有科学边界完成对应流程。
